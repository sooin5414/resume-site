PROJECT #1

Automaed Grammer Explanatiopn System for Korean Language Learners

외국인 대상 한국어 문법 설명 자동 생성 시스템

프로젝트 개요

외국인 학습자에게 한국어 문장 구조를 설명하기 위한 자동 시각화 시스템

형태소분석 ~ 구문분석까지 파이프라인을 구축하여 3단계 Bar Chart를 1초 만에 생성

🔴 문제 정의

  • 언어학 전공자가 한 문장에 분석에 최소 20분 소요
  • 형태소 →  품사 →  구문 분석  →  시각화 수작업 필요 
  • 교육 콘첸츠 제작 효율 저하 

🟢 프로젝트 목표

  • 형태소 기반 구문분석기를 활용한 Span 단위 자동 추출 엔진 개발
  • 교육용 문법 설명에 즉시 활용 가능한 3단계 바 차트 자동 생성
  • 처리속도 20분 → 1초(1200배)

📋 기술 상세

1
형태소 분석 (Morpheme Segmentation)
📌 기술적 배경
• 어절 단위의 한계
조사/어미가 어절에 결합되어 세밀한 구문 관계 분석 불가
• 형태소 단위 처리의 필요성
KLUE-DP(데이터셋)를 형태소 단위로 변환하여 활용 
     기존: 어절 단위 의존 구문 분석 
     개선: 형태소 단위로 변환 후 세밀한 문법 분석 가능
🔧 기술적 특징
• Kiwi 형태소 분석기
한국어 어절을 형태소 단위로 분할
예: "장마철이라" → ["장마철"/NNG, "이"/VCP, "라"/EC]
• 높은 분할 정확도
대규모 말뭉치 기반 학습 모델로 신뢰성 확보
2
품사 태깅 & 영어 Glossing
🎯 핵심 목적 (왜 BERT를 학습시켰는가?)
• Kiwi 형태소 분석기 출력 형태소와 품사를 그대로 사용하지 않음
Kiwi는 형태소 분할은 잘 하지만, 품사 태깅에서 문맥을 잘 고려하지 못함
   예) 복합 형태소, 신조어/전문용어 
동일 형태소가 문맥에 따라 다른 품사로 쓰이는 중의성 문제 발생
  → 학습 후, 96.36% 달성(KLUE_DP dev set기준)
🔄 BERT 재처리 과정(Kiwi →BERT 재처리 → 최종 품사 태그)
① Kiwi 결과(형태소 + 품사)를 BERT 입력으로 변환 
② Self-attention으로 전체 문장 맥락 파악 
③ 각 형태소에 대해 43개 세종 품사 중 문맥상 가장 적합한 태그 선택 
④ 최종 품사 태그 출력 (Kiwi 품사 교정 완료)
🌐 영어 뜻(Glossing) 태깅
 • RoBERTa Cross Encoder fine-tunning(문맥 기반 최적 gloss 선택)
   형태소-품사-gloss pair 구축 후 fine-tunning 
 여러 gloss 후보 중 문맥에 가장 적합한 것을 스코어링(다의어의 문맥에 맞는 정확한 의미 선택)
   예: "배" → 문맥 분석 → "ship" / "stomach" / "pear" 중 선택
3
SuPar 기반 형태소 단위 구문 분석
📖 의존 구문 분석이란?
문장 내 단어(형태소) 간의 문법적 관계를 파악하는 작업
• 각 형태소가 어느 형태소에 의존하는지 (head)
• 어떤 문법적 관계인지 (relation: 주어, 목적어 등)
🎯 KLUE-DP 의존 관계 체계 선택
• 선택 이유
UD(Universal Dependencies): 다국어에서 많이 쓰이지만, 한국어 특수성(조사·어미 결합) 반영 제한적 
  → KLUE-DP 선택: 
 • 한국어 특화: 조사, 어미 등 한국어 고유 문법 요소를 세밀하게 구분  
 • 형태소 단위: 교착어인 한국어 특성에 맞는 형태소 단위 분석  
 • 풍부한 기능 태그: NP_SBJ(주어), NP_OBJ(목적어), NP_AJT(부가어) 등 36종 관계
🏗️ SuPar 모델 아키텍처
입력: 형태소 시퀀스 + 품사 태그
↓
KLUE RoBERTa Encoder (문맥 임베딩)
↓
POS Tag Embeddings (품사 정보 결합)
↓
LSTM Encoder (순차 정보 인코딩)
↓
Biaffine Attention (의존 관계 예측)
↓
출력: 의존 관계 (head, relation)
🌲 의존 구문 분석 트리
형태소 단위 의존 관계 → ROOT를 중심으로 문법적 역할 식별
어요 (EF) [root] ├─ [VP] 내리 (VV) │ ├─ [VNP] 라 (EC) │ │ ├─ [NP_AJT] 요즘 (NNG) (these days) │ │ ├─ [morph] 장마철 (NNG) (rainy season) │ │ └─ [morph] 이 (VCP) (be) │ ├─ [NP_SBJ] 가 (JKS) │ │ └─ [morph] 비 (NNG) (rain) │ └─ [AP] 많이 (MAG) (a.lot)
4
구/절 스팬 추출
🎯 스팬 추출 방식
SuPar의 의존 구문 분석 트리와 형태소 태그를 활용하여 Rule-based 알고리즘으로 문장의 의미 단위(구/절)를 자동으로 추출
• 구(Phrase): 주어구, 목적어구, 부사구 등
• 절(Clause): 주절, 종속절, 관형절 등
📊 추출 결과 예시
문장: "요즘 장마철이라 비가 많이 내려요"

구(Phrase) 추출:
• 부사구(Adverb): [요즘]
• 서술구(Verb Phrase): [장마철, 이, 라]
• 주어구(Subject Phrase): [비, 가]
• 부사구(Adverb): [많이]
• 서술구(Verb Phrase): [내리, 어요]

절(Clause) 추출:
• 종속절(Subordinate): [요즘, 장마철, 이, 라]
• 주절(Main Clause): [비, 가, 많이, 내리, 어요]

🔹 Agent 관점 설계 및 반복 개선 구조

✔️ 설계 배경 (Why Agent-style?)
본 시스템은 현재 rule-based 파이프라인으로 구현되어 있지만,
한국어 문법 분석 특성상 단일 실행 결과를 확정하기 어려운 문제 구조를 가짐
✔️ 문제의 본질 (Why Loop is Needed)
형태소 분절 / 품사 태깅 오류
→ 구문 분석 오류 
→ 구·절 추출 오류 
→ 문법 설명 품질 저하로 연쇄 전파 
 한 번의 분석으로 종료되는 구조는 본질적으로 한계 존재
✔️ 핵심 설계 판단 (Decision Point 정의)
• 형태소 분절 및 품사 태깅을 시스템의 핵심 decision point로 정의
• 상위 단계(문법 설명) 결과를 하위 분석 단계의 피드백 신호로 활용 가능하도록 설계
🔄 Agent-style 반복 개선 구조 (개념)
1. 형태소 분절 및 품사 태깅 수행 2. 형태소 단위 구문 분석 및 구/절 추출 3. 문법 설명 시각화 생성 4. 설명 품질 및 구조적 일관성 평가 5. 품질 기준 미달 시:
• 형태소 분절 / 품사 태깅 결과 재검토
• 수정된 분석 결과로 파이프라인 재실행
6. 설명 품질이 수렴할 때까지 반복

현재 한계 &  내가 제안한 해결책

❌ 문제: Rule-based 방식은 복잡한 문장에서 정확도 저하
✅ 해결책: Semi-supervised Learning 전략
1) Rule-based로 10,000개 자동 생성
2) 1,000개 수동 검수하여 Gold Data 구축
3) BIO 태깅 기반 Token Classification 모델 학습
💡 기대 효과
• 복잡한 문장 구조에서 정확도 대폭 개선 예상 
• 수작업 대비 90% 비용 절감 (1,000개 검수 vs 10,000개)
• 학습된 모델로 새로운 문장 패턴에 대한 자동 학습 가능

PROJECT #2 (석사 논문)

Genetic Algorithm-based RAG Re-ranker Optimization

유전 알고리즘 기반 RAG 리랭커 검색 최적화 시스템

프로젝트 개요

금융 재무제표 RAG 시스템에서 6가지 Re-ranker를 유전 알고리즘(GA)으로 최적 조합하여 검색 정확도를 2.54% → 76.69%로 개선했습니다. 

✅ SOTA 단일 모델 대비 +3.81%p 향상 

✅ Re-ranker 미사용 대비 +74.15%p 향상 

✅ 데이터셋: FinRAG (금융 재무제표 이미지 기반 QA 벤치마크) - 893개 문서, 708개 질의응답

🔴 해결한 문제

  • 단순 Vector DB 검색 → 정확도 2.54% (거의 사용 불가)
  • 단일 Re-ranker 모델 → 복잡한 쿼리 대응 한계
  • 고정 가중치 방식 → 쿼리 특성별 최적화 불가
  • 예시: "2023년 Revenue는?" → 엉뚱한 문서 검색 → LLM이 틀린 답변 생성

💡 핵심 인사이트 & 배운 점

✅ 앙상블 > 최고 단일 모델
FlashRank(SOTA) 대비 +3.81%p 향상. 각 모델의 장점을 결합하여 안정적 성능 확보
✅ 경량 모델 조합의 효율성
BM25(무료) + Bi-Encoder + FlashRank 조합으로 비용 효율적이면서도 높은 정확도 달성
✅ Re-ranker의 필수성 입증
Re-ranker 없으면 2.54% (거의 사용 불가) → 사용 시 최소 16.24% 이상
✅ GA 최적화의 효과
1,000세대 진화 알고리즘으로 최적 가중치 자동 탐색. 수동 튜닝 대비 시간 절약 + 성능 향상
📊 가중치 분석 결과
• FlashRank: 0.937 (가장 높은 기여도) • TF-IDF: 0.656 / Bi-Encoder: 0.637 • Cross-Encoder, BM25, Cohere: 상황별 보완 역할 → FlashRank가 핵심, 나머지 모델이 약점 보완
